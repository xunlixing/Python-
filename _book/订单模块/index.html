订单模块 Order

下单

/**
*  user_id:用户ID
*  designer_id:设计师ID
*  remark:订单备注
*  products:商品列表 [{购物车ID,商品ID,购买数量}]
*  address:收货人信息 {收货人姓名,收货人手机号,收货地址国家标识,收货详细地址,邮编}
*  return:Object格式见下面
*/
Object create(user_id，designer_id，remark，products，address){ 
    - 验证设计师运费设置，并返回运费:
    - 验证商品列表中购物车ID是否都存在，都存在方可下单。
    - 验证商品状态
    - 验证商品库存
    - 从库中获取商品单价
    - 计算订单金额
    - 生成订单号
    - 插入订单表、插入订单商品表、移除购物车商品列表(事务处理)
}

Object格式

{
  exception:"OrderExceptionType"/null
  data:数据
}

实现处理说明

  • 处理逻辑验证不通过，直接return下面设置的Exception类型，data为null；
  • 处理逻辑验证通过，exception设置为null，data为{订单号,订单金额,运费}

异常类型 OrderExceptionType

  • 商品下架：ProductUnderException
  • 库存不足：ProductInventoryException
  • 部分商品已下单：OrderRepeatException
  • 货运国家不匹配：ShipException
  • 其他异常统一为： OrderException

3.支付成功后更新订单

说明：支付成功通知处理

/**
*  Object data:
*     - order_no:订单号
*     - trade_no:支付交易订单号
*     - channel:支付渠道
*     - currency:币种
*     - amount:实际支付金额
*     - business_account:平台收款账号
*     - customer_account:用户付款账号
*/
void updateWithPaymentSuccess(Object data){
        1. 处理支付表重复写入：
            - 依据渠道和交易订单号验证支付表交易号在否存在，没有继续执行
        2. 验证订单状态：
            - 如果状态不是未支付，则只写入支付表
            - 如果状态是未支付，则继续往下处理
        3. 验证商品列表库存：
            -商品库存不足：
                   - 写入支付表
                - 更新订单表状态、时间、回写支付交易号
                - 写入退单表：(退单原因：库存不足；退单类型：退款)
                - 写入退款表
            -商品库存充足：
                - 写入支付表
                - 更新订单状态、时间、回写支付交易号
                - 扣减库存
}

发货

/**
*  order_id:订单标识
*  logistics_no:物流单号
*  logistics_company:物流公司
*/
Object updateWithDeliver(order_id,logistics_no,logistics_company){
    - 验证订单状态：只有已支付才可以发货
    - 验证是否有退单(包含退单和退单申请)
    - 更新订单表物流信息、订单状态、时间
}

Object格式

{
  exception:"OrderExceptionType"/null
  data:数据
}

实现处理说明

  • 处理逻辑验证不通过，直接return下面设置的Exception类型，data根据实际情况处理；
  • 处理逻辑验证通过，exception设置为null，data为TRUE

异常类型 OrderExceptionType

  • 订单状态不一致：StateDifferenceException —>data:当前订单状态
  • 其他异常统一为： OrderException

退单

说明：后台调用

1. 退单申请

/** 
*  order_id:订单标识 
*  type:退单类型
*  reason:退单原因
*  return：
*/
Bool createRevert(order_id,type,reason){
      - 验证订单状态：只有已支付和已收货且无退单申请
    - 写入退单表
       -根据退单类型，写入金额：
          退款：订单金额+运费
          退货：订单金额 
}

2. 退单

说明：退款、退货公用一个方法

/** 
*  revert_id:退单标识 
*  logistics_no:物流单号
*  logistics_company:物流公司
*  return：
*/
Bool agreeRevert(revert_id,logistics_no,logistics_company){
    1. 验证退单状态：处理过直接返回true
      2. 根据revert_id获取退单类型
          退款：继续执行
          退货：验证 logistics_no,logistics_company 非空
    3. 更新退单表
    4. 更新订单状态
    5. 写入退款表
    注：3、4、5事务处理
}

3. 驳回

Bool rejectRevert(revert_id,reason){
    - 验证退单表状态：处理过直接返回true
    - 更新退单表
}
